In my breakdown, I specified what happens with my GameController, specifically how it now works with my state machine. The wall item now tells the GameController in my breakdown that, once the win condition is achieved, the quest status is updated to Complete, which in turn, within the GameController, should update the new currentQuest to set up a new one. I also added within my GameController section that if the status is Ongoing, that the timer, which I added to the attributes parts of the section, would go down. Then, I showed that once the timer hits 0, the fail screen is activated, which has a restart button that resets the scene, which I added to my representation through adding a big bubble representing the entire scene. Going into the specifics of how my state machine works, essentially everything runs within my GameController script. In my script, I added a timer variable that goes down in update only if the questStatus variable is Ongoing, with all my text also going through. If the questStatus variable is Failed, all the text onscreen is disabled and my FailScreen game object is enabled, which has a button that resets the scene. Finally, if the questStatus variable is Complete, the questStatus is set back to Ongoing so that the code doesn't keep looping as the following code after this sets the currentQuest to the nextQuest QuestNode variable, setting a new quest for the player to follow. The way each of these statuses are triggered is that Ongoing is the default quest setting set at the Start method or when a new quest is set from if the questStatus is Complete. The way the Failed status is triggered is when the timer variable kept track of within the GameController script hits 0, meaning the player has run out of time to complete the quest and thereby failed it. The Complete status can only be triggered currently by outside scripts or classes, with my only one currently being able to do so is my goal platform, but the format should be the same for other win class objects like my wall in my breakdown. Once the win condition I want my player to achieve is completed, in this case being once the player hits the goal platform, the script attatched to the win condition object accesses the GameController script and sets the questStatus variable to Complete.
